# 날자 아카이브 편집형 일러스트 홈 개편 설계

작성일: 2026-06-18
대상: GitHub Pages의 날자 아카이브 블로그 홈

## 1. 목적과 범위

날자 아카이브 홈을 손그림과 넓은 여백이 어우러진 편집형 페이지로 개편한다. 참고 이미지의 오프화이트 종이 질감, 먹색 선, 옅은 민트와 제한된 파스텔 포인트를 가져오되 날자 아카이브의 콘텐츠와 독자에게 맞게 새로 구성한다.

이번 작업은 홈을 먼저 개편하고 검증하는 1단계다. 글 상세, 소개, 자료 원칙, 질문 안내 페이지는 홈의 결과를 확인한 뒤 같은 시각 시스템으로 확장한다.

현재 정보 구조, Jekyll 글 데이터, 주요 탐색 링크는 유지한다. 이메일 구독 기능은 추가하지 않는다.

## 2. 승인된 디자인 방향

### 전체 인상

- 종이 위에 정돈된 인쇄 편집물 같은 분위기
- 넓은 여백과 중앙 정렬된 명조 제목
- 새까맣지 않은 먹색 선과 텍스트
- 따뜻한 오프화이트 배경
- 옅은 세이지 민트를 주조색으로 사용
- 연분홍, 연노랑, 하늘색은 일러스트의 작은 부분에만 사용
- 그림자와 둥근 카드 사용은 크게 줄임

현재 사이트의 쨍한 주황색은 필수 브랜드 색으로 취급하지 않는다. 새 홈에서는 참고 이미지에 가까운 차분한 색조를 우선한다.

### 타이포그래피

- 큰 제목과 섹션 제목: 한글 명조 계열
- 본문, 메뉴, 메타데이터: 읽기 쉬운 고딕 계열
- 영문 소제목은 작고 넓은 자간으로 사용
- 긴 본문이나 작은 화면에서 명조체를 과도하게 사용하지 않음

제목은 `Noto Serif KR` 500·600 굵기를 사용하고 `AppleMyungjo`, `Batang`, 기본 serif를 순서대로 대체 글꼴로 둔다. 본문과 메뉴는 현재의 Pretendard·시스템 고딕 스택을 유지한다. 웹 폰트가 늦거나 실패해도 텍스트가 즉시 표시되도록 `font-display: swap` 동작을 사용한다.

## 3. 홈 정보 구조

홈은 다음 순서로 구성한다.

1. 최소한의 헤더
   - 날자 아카이브 브랜드
   - 글, 소개, 자료 원칙, 질문 안내
   - 도서출판 날자 외부 링크
2. 히어로
   - `배움과 선택을 잇는 기록` 소제목
   - `배운 것이 삶으로 이어지도록` 핵심 제목
   - 아카이브 소개 문장
   - `최신 글 읽기` 버튼
   - 여러 사람의 배움이 이어지는 메인 손그림
3. 최근 아카이브
   - 최신 글 한 편을 큰 편집형 카드로 강조
   - 이어지는 글을 작은 목록 또는 카드로 배치
4. 세 갈래의 질문
   - 유추와 문해력
   - 배움과 일상
   - 자료와 실천
   - 각 항목에 작은 선화 SVG 아이콘 사용
5. 편집 원칙 배너
   - `쉽게 쓰되, 근거는 가볍게 다루지 않습니다` 메시지
   - 자료 원칙 페이지 링크
   - 편집자 또는 자료를 읽는 인물의 보조 손그림
6. 마지막 행동 영역
   - 이메일 입력창 대신 최신 글과 도서출판 날자 페이지로 연결
7. 최소한의 푸터

## 4. 일러스트 시스템

### 메인 일러스트

메인 손그림은 학습자, 보호자, 교사 또는 실무자가 각자의 공간에서 배우고 읽고 대화하는 여러 장면을 하나의 흐름으로 연결한다. 특정 인물 하나를 수혜자로 고정하지 않고, 배움에 참여하는 여러 주체를 동등하게 보여준다.

표현 원칙:

- 발달장애 청소년과 성인을 아동화하지 않음
- 과도하게 귀엽거나 유아적인 표정과 비율을 피함
- 인물의 역할을 고정된 성별이나 외형으로 구분하지 않음
- 얼굴, 목, 손에는 피부색 면 채움을 사용하지 않고 배경과 같은 종이색을 유지해 특정 인종으로 읽히지 않게 함
- 눈, 코, 입은 현재의 간결한 검은 선 표현을 유지하고 인물의 표정과 동작은 보존함
- 가는 먹색 선과 제한된 면색을 사용
- 참고 이미지의 구도를 복제하지 않고 날자 아카이브의 메시지에 맞는 새 장면을 제작

### 제작 방식

혼합형 에셋 전략을 사용한다.

- 메인 장면과 편집 원칙 배너: 완성도 높은 투명 배경 일러스트 이미지
- 주제 아이콘과 작은 장식: 동일한 선 굵기의 SVG
- 투명 일러스트는 WebP를 우선 제공하고 PNG를 `<picture>` 대체 소스로 둠
- 데스크톱 표시 폭의 약 2배 크기로 내보내 고해상도 화면에서도 선이 흐려지지 않게 함
- 순수 장식 이미지는 빈 `alt` 또는 CSS 배경으로 처리

## 5. 컴포넌트와 데이터 흐름

기존 Jekyll 구조를 유지한다.

- 홈은 `site.posts`를 사용해 최신 글을 날짜순으로 렌더링한다.
- 첫 글은 강조 영역에 배치하고, 이후 글은 보조 목록으로 배치한다.
- 글 제목, 설명, 분류, 날짜, 읽는 시간은 기존 front matter와 include 계산을 재사용한다.
- 새 구독 저장소, API, 폼 처리, 클라이언트 상태는 만들지 않는다.
- 링크는 기존 `relative_url` 방식을 유지해 GitHub Pages 프로젝트 경로에서도 동작하게 한다.

홈 전용 마크업과 스타일은 기존 레이아웃의 접근성 계약을 깨지 않는 범위에서 추가한다. 반복되는 홈 요소가 두 곳 이상에서 필요해질 때만 include로 분리한다.

## 6. 반응형 동작

### 데스크톱

- 히어로 제목과 설명을 중앙에 둔다.
- 메인 일러스트는 여러 인물 프레임이 가로로 이어지는 구성으로 표시한다.
- 최근 글은 한 개의 큰 항목과 작은 보조 항목으로 대비를 만든다.
- 주제 세 갈래는 세 열로 표시한다.

### 모바일

- 현재처럼 메뉴를 두 줄까지 자연스럽게 줄바꿈하되, 외부 링크 버튼을 마지막 줄에 독립적으로 배치한다.
- 히어로 제목 크기와 줄 수를 줄인다.
- 여러 인물 장면을 겹치지 않는 단순한 세로 또는 축약 구도로 바꾼다.
- 최근 글과 주제 목록은 한 열로 표시한다.
- 모든 버튼과 링크의 터치 영역을 충분히 확보한다.

## 7. 접근성과 실패 대응

- 키보드로 모든 링크에 접근할 수 있어야 한다.
- 포커스 표시를 배경과 구분되는 색으로 유지한다.
- 본문과 배경, 버튼과 배경의 명암 대비를 확인한다.
- 색상만으로 글 분류나 링크 상태를 전달하지 않는다.
- 장식 손그림은 스크린리더가 건너뛰게 한다.
- 정보를 전달하는 이미지가 생기면 구체적인 한국어 대체 텍스트를 제공한다.
- 이미지가 로드되지 않아도 제목, 소개, 최신 글 링크, 주제 설명이 온전히 남는다.
- `prefers-reduced-motion`을 존중하며, 핵심 정보를 애니메이션에 의존하지 않는다.

## 8. 검증 기준

홈 개편은 다음 조건을 모두 만족해야 완료로 본다.

1. Jekyll 프로덕션 빌드가 오류 없이 완료된다.
2. 기존 `scripts/verify_site.rb` 검증이 통과한다.
3. GitHub Pages의 `baseurl`이 적용된 모든 내부 링크와 이미지 경로가 동작한다.
4. 최신 글의 강조 순서가 `site.posts` 날짜순과 일치한다.
5. 데스크톱과 모바일에서 텍스트, 일러스트, 카드가 겹치거나 잘리지 않는다.
6. 이미지 로드 실패 상태에서도 핵심 콘텐츠와 탐색이 가능하다.
7. 키보드 탐색, 포커스 표시, 제목 순서, 장식 이미지 처리가 접근성 원칙을 만족한다.
8. 홈의 새로운 색과 글꼴이 충분한 명암 대비와 읽기 성능을 제공한다.

## 9. 이번 단계에서 하지 않는 일

- 글 상세와 일반 페이지의 전면 개편
- 이메일 뉴스레터 구독 기능
- 검색, 댓글, 다국어 기능
- Jekyll 콘텐츠 모델 또는 배포 방식 변경
- 참고 이미지 속 기존 일러스트의 복제

홈 개편 검증 후 2단계에서 글 상세와 일반 페이지를 같은 시각 시스템으로 확장할지 결정한다.
