# 「쉬운 글만으로 충분하지 않은 이유」 게시 설계

## 목표

제공된 원고 `09-why-easy-text-alone-is-not-enough.md`를 날자 아카이브의 기존 게시물 형식과 편집 원칙에 맞춰 한 편의 새 글로 게시한다. AI 선언문은 홈페이지 왼쪽 민트 박스에 계속 고정하고, 새 글은 오른쪽 일반 글 목록의 맨 위에 표시한다.

## 원문 보존과 편집 원칙

- 제목 `쉬운 글만으로 충분하지 않은 이유`와 핵심 주장·문단 구조를 유지한다.
- 공개 문구의 `발달장애인`, `발달장애 성인`, `발달장애인을`은 문맥에 맞게 `지적장애인`, `지적장애 성인`, `지적장애인을`로 통일한다.
- 원문의 의미를 확장하거나 효과를 보장하는 문장을 새로 만들지 않는다.
- 중복 렌더링을 막기 위해 본문의 최상위 `# 제목`은 제거한다. 제목은 게시물 레이아웃이 front matter에서 렌더링한다.
- 레이아웃이 제공하는 작성 기준 안내와 겹치는 원문 하단 `편집 기준` 섹션은 제거한다.

## 게시물 파일과 메타데이터

- 새 파일은 `_posts/2026-06-19-why-easy-text-alone-is-not-enough.md`로 만든다.
- URL은 `/naljabooks-blog/archive/why-easy-text-alone-is-not-enough/`로 생성한다.
- `layout`은 `post`, `date`와 `updated`는 모두 `2026-06-19`로 설정한다.
- 작성자는 기존 스키마에 맞춘 Organization 객체 `도서출판 날자 · 날자꾸러미 편집부`로 기록한다.
- 카테고리는 기존 홈페이지 분류와 어울리는 `문해력과 쉬운 정보`로 정리한다.
- 태그는 `쉬운 글`, `읽기이해`, `문해력`, `지적장애`, `학습설계`를 사용한다.
- 원문의 SEO 키워드는 별도 미사용 front matter 필드로 남기지 않고 제목·설명·본문·태그에 자연스럽게 반영한다.

## 목차와 본문 구조

- 핵심 요약과 1~6절, 결론을 기존 게시물의 2단계 제목으로 유지한다.
- 각 주요 절에 영문 소문자 ID를 지정하고 front matter `toc`와 일치시킨다.
- 목차 순서는 `핵심 요약`, `쉬운 글의 역할`, `이해의 조건`, `이해 활동`, `성인성 존중`, `반복과 피드백`, `날자꾸러미의 관점`, `결론`이다.
- 본문의 첫 문단은 원문의 결론형 요약을 유지한다.

## 출처

- 원문의 참고자료 네 개를 front matter `sources` 배열로 옮긴다.
- 각 출처에 제목, 기관 또는 저자, URL을 기록한다.
- 관련 주장을 설명하는 본문 문장에 `[1](#source-1)` 형식의 출처 링크를 연결한다.
- 본문 하단의 원문 `참고자료` 목록은 제거하고 레이아웃의 통합 출처 목록을 사용한다.

## 홈페이지 표시

- AI 선언문의 `pinned: true`와 왼쪽 `featured-story` 위치를 유지한다.
- 새 글은 `pinned`가 없는 일반 게시물이므로 오른쪽 `story-list` 첫 항목에 표시한다.
- 기존 일반 글 `지적장애인에게 왜 유추력이 필요할까?`는 새 글 다음에 유지한다.

## 검증

- production Jekyll 빌드가 성공해야 한다.
- 새 글 페이지에 고유 title, description, canonical, Open Graph, 게시·수정 시각, BlogPosting JSON-LD가 생성되어야 한다.
- 목차의 모든 링크가 실제 본문 ID와 대응해야 한다.
- 출처 네 개가 통합 출처 목록에 렌더링되어야 한다.
- 새 글 공개 HTML과 공개 원본에 `발달장애` 표현이 없어야 한다.
- `sitemap.xml`과 `feed.xml`에 새 글 URL이 포함되어야 한다.
- 홈페이지에서 선언문 링크는 왼쪽에 한 번만 유지되고, 새 글은 오른쪽 첫 항목이어야 한다.
- 미래 게시물 통합 검증 `scripts/verify_pinned_home.rb`가 계속 통과해야 한다.
